Abstract
In this paper, we present an HW version of the OpenMax (OMX) standard Core component, which represents the bulk of the OMX Integration Layer (IL). The OMX Core is responsible of the communications between the components that implement the functionality; this is a key aspect in any multimedia application since the efficiency of the methods to move data from one processing element to another determines the throughput of the system. Our OMX Core HW version supports, among many other aspects, tunneled and non-tunneled communications, port reconnection and dynamic management of buffers and their associated resources.
